# Связи между сущностями

## Sales и Products

**Тип связи:** M:1 
**Объяснение:** Множество записей продаж (sales) могут ссылаться на один и тот же product_id. Одна запись в sales относится только к одному продукту.

## Employees и Cities

**Тип связи:** M:1 
**Объяснение:** Множество сотрудников могут проживать в одном городе. Один сотрудник не может проживать в нескольких городах одновременно.

## Cities и Countries

**Тип связи:** M:1 
**Объяснение:** Множество городов находятся в одной стране. Один город не может находиться в нескольких странах.

## Все связи

**countries → cities**
- Тип: 1:M 
- Одна страна имеет много городов

**cities → shops**
- Тип: 1:M 
- Один город имеет много магазинов

**shops → employees**
- Тип: 1:M 
- Один магазин имеет много сотрудников

**employees → sales**
- Тип: 1:M 
- Один сотрудник обслуживает много продаж

**customers → sales**
- Тип: 1:M 
- Один покупатель совершает много покупок

**products → sales**
- Тип: 1:M
- Один продукт продается много раз

**categories → products**
- Тип: 1:M 
- Одна категория содержит много продуктов

**cities → employees**
- Тип: 1:M 
- В одном городе может проживать много сотрудников

**cities → customers**
- Тип: 1:M 
- В одном городе может проживать много покупателей
