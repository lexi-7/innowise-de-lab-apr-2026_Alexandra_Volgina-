# Ограничения для таблиц sales и products

## Sales

**NOT NULL:**

- employee_id - продажа должна быть привязана к сотруднику
- customer_id - продукт кто-то купил
- product_id - продукт кто-то купил
- quantity - количество не может быть неизвестно
- total_price - итоговая цена нужна
- sales_timestamp - время продажи не может не существовать

**UNIQUE:**

- sales_id - первичный ключ, он уникальный
- transaction_number - номер транзакции должен быть уникальным

**CHECK:**

- quantity > 0 - нельзя продать 0 или отрицательное количество
- discount >= 0 and discount <= 100 - скидка от 0% до 100%
- total_price >= 0 - цена не может быть отрицательной

## Products

**NOT NULL:**

- product_id - у каждого товара должен быть ID
- product_name - товар должен как-то называться
- price - должна быть цена
- category_id - товар должен относиться к какой-то категории

**UNIQUE:**

- product_id - первичный ключ
- product_name - названия продуктов лучше не повторять

**CHECK:**

- price > 0 - цена должна быть больше 0
- vitality_days >= 0 - срок годности не может быть отрицательным
- class in ('A', 'B', 'C') - только определённые классы
