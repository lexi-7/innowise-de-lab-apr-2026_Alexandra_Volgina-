# Логическая модель данных

## Список сущностей

1. **countries** (страны)
2. **cities** (города)
3. **shops** (магазины)
4. **employees** (сотрудники)
5. **customers** (покупатели)
6. **categories** (категории)
7. **products** (продукты)
8. **sales** (продажи)

## Иерархические связи

**countries → cities**
- Тип связи: 1:M
- Одна страна может содержать много городов

**cities → shops**
- Тип связи:1:M
- Один город может содержать много магазинов

**shops → employees**
- Тип связи: 1:M
- Один магазин может нанимать много сотрудников

## Связи продаж

**employees → sales**
- Тип связи: 1:M
- Один сотрудник обслуживает много продаж

**customers → sales**
- Тип связи: 1:M
- Один покупатель совершает много покупок

**products → sales**
- Тип связи: 1:M
- Один продукт продается много раз

## Связи категорий

**categories → products**
- Тип связи: 1:M
- Одна категория содержит много продуктов

## Связи проживания

**cities → employees**
- Тип связи: 1:M
- В одном городе может проживать много сотрудников

**cities → customers**
- Тип связи: 1:M
- В одном городе может проживать много покупателей

## Итоговая таблица связей (текстовый формат)

- countries (1) → (M) cities
- cities (1) → (M) shops
- shops (1) → (M) employees
- employees (1) → (M) sales
- customers (1) → (M) sales
- products (1) → (M) sales
- categories (1) → (M) products
- cities (1) → (M) employees (проживание)
- cities (1) → (M) customers (проживание)

## Примечания

- Все связи в модели являются типом "один-ко-многим" (1:M)
- Связи "многие-ко-многим" (M:N) в модели отсутствуют
- Связи "один-к-одному" (1:1) в модели отсутствуют
- Таблица `sales` является фактической (хранит события продаж)
- Остальные таблицы являются справочниками
